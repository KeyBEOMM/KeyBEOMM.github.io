# 23강 VLA (Vision-Language-Action Model) 강의 정리본

---

## 0. 강의 한눈에 보기

**강의자료가 잡은 큰 그림 (도입부 프레이밍, p1–7):**
- 로봇 시대의 도입 → **"지능은 어디서 오는가"**라는 큰 질문에서 출발. 자연(생물의 움직임에서 지능이 발생)과 인공지능(데이터로 지능을 구현)이 **반대 방향으로 진화하지만 결국 하나의 목적(Embodiment, 로봇에 지능 통합)에 도달**한다는 프레이밍.
- 로봇 제어를 **Perception → Planning → Action** 세 단계로 분해하고, **전통적 접근(각 단계를 따로 개발 후 통합)** vs **Foundation Model 접근**을 대비.
- 핵심 질문(p7): **"인터넷 스케일 데이터를 로보틱스에 활용해 generalist policy를 얻을 수 있는가?"** 그리고 **"제어 루프(control loop) 중 얼마만큼을 Foundation Model이 담당할 것인가?"**
- 이 질문에 대한 **두 가지 전략**이 강의 전체의 뼈대:
  1. **로보틱스 전용 Foundation Model을 새로 만든다 → VLA**
  2. **사전학습된 Foundation Model을 high-level planner로 쓴다 → VLM + (별도 제어 모듈)**

**다룬 개념(모델) 목록 및 중요도:**

| # | 개념 / 모델 | 중요도 |
|---|------|--------|
| 1 | 도입 — 지능의 발생, 로봇 제어 3단계, 전통적 접근 | ★ |
| 2 | Foundation Model의 등장과 VLA의 동기 (2전략) | ★★★ |
| 3 | **RT-1** (Google) — 최초의 본격 로보틱스 트랜스포머 | ★★★ |
| 4 | RT-1 핵심 부품: FiLM / USE / TokenLearner / Action 토큰화 | ★★★ |
| 5 | **PaLM-E** — Embodied LLM (단, VLA가 아님!) | ★★ |
| 6 | **RT-2** — 웹 지식을 제어로 전이, "VLA"라는 이름 등장 | ★★★ |
| 7 | RT-2-X & Open-X-Embodiment 데이터셋 | ★★ |
| 8 | **OpenVLA** — 오픈소스 7B VLA (DINOv2+SigLIP) | ★★ |
| 9 | **Octo** — diffusion action head, easy-to-finetune | ★★ |
| 10 | **π₀ (pi-zero)** — flow matching, action sequence 동시 예측 | ★★ |
| 11 | 결론: 성과·한계·과제 | ★ |
| 12 | 대논쟁: **VLA vs VLM+Control** | ★★ |

**강의의 위치:**
- 흐름상 Imitation Learning(19·22강)에서 다룬 "데모로부터 정책 학습"의 연장선이자, 24강 Offline RL·25강 World Model로 이어지는 **로보틱스·Foundation Model 묶음**의 한 축.
- VLA는 본질적으로 **언어 조건부 행동 복제(language-conditioned behavior cloning)**의 대규모 버전이라, IL 강의와 직접 연결된다. `[보충]`
- 다음: Offline RL(24강) / World Model(25강, 학기 종료).

## 1. 도입 — 지능의 발생과 로봇 제어의 구조 ★

**자료의 설명 흐름 (p2–5):**
1. 로봇 시대의 초기 영상으로 시작.
2. **지능의 발생(Emergence of Intelligence, p3):** 자연에서는 "생물의 움직임 → 복잡한 센싱 시스템 → 지능 발달"이라는 순서로 지능이 진화했고, 인공지능은 거꾸로 "데이터로 지능을 구현 → 그 지능을 신체(로봇)에 통합"하는 방향. **두 방향이 결국 한 점(Embodied Intelligence)에서 만난다**는 것이 핵심 메시지.
3. **로봇 제어 3단계(p4):**
   - **Perception**: 센서(카메라·라이다·마이크)로 환경 정보 수집
   - **Planning**: 어디로 갈지, 무엇과 상호작용할지, 어떻게 task를 달성할지 결정
   - **Action**: 액추에이터(모터·그리퍼·바퀴)로 계획 실행
4. **전통적 접근(p5):** 각 task를 **따로 개발한 뒤 통합**.
   - Perception: Sensor Fusion/Calibration, Detection·Tracking, 6DOF Pose estimation
   - Planning: Localization, Path Planning, Grasping Strategy
   - Action: Motor Control, Gripper Actuation, Feedback Control
   - → 이렇게 모듈을 따로 만들어 **integration**하는 방식. (이 "모듈 분리"가 §12 VLA vs VLM+Control 논쟁의 출발점이 된다.)

**중요도 근거:** ★ — 빠르게 배경을 깔고 지나가는 도입부. 단, "제어를 3단계로 나눈다"는 틀은 이후 **"Foundation Model이 이 루프의 어디까지를 먹느냐"**(p7)로 직결되므로 맥락상 중요.

## 2. Foundation Model의 등장과 VLA의 동기 ★★★

**자료의 설명 흐름 (p6–7):**
**Foundation Model(p6)** 정의: 방대하고 다양하며 종종 레이블 없는 데이터(텍스트·이미지·코드·멀티모달)로 **사전학습된 모델**. 다음의 연쇄로 거대 모델이 가능해졌다고 정리:
- **Transformer의 등장**(GPT-1) → 대형 모델의 구조 제공
- **인터넷 데이터** → 대형 모델 학습 연료
- **Scaling Laws** → 데이터 기반 거인의 성능 우위
- **확장**: NLP → 멀티모달 → 그 너머
- **Frontier(최전선): 로봇 액션을 생성할 수 있는가?**

> 핵심 도식(p6): **Transformer(GPT-1) + Large Data = Large Language(Vision) Model** → 이 공식을 로봇으로 확장하면?

**VLA의 동기와 두 전략 (p7) — 강의 전체의 뼈대:**
- **Motivation:** 인터넷 스케일 데이터를 로보틱스에 활용해 **generalist policy**(하나로 여러 task를 하는 정책)를 얻을 수 있는가?
- **Key Issue:** Foundation Model이 **control loop의 얼마만큼을 담당**할 것인가?
- **두 가지 전략:**
  1. **VLA**: 로보틱스 전용 Foundation Model을 새로 만든다 → perception·planning·action을 **하나의 모델이 end-to-end로** 담당.
  2. **VLM + ( )**: 사전학습된 VLM을 **high-level planner**로 쓰고, 실제 action은 별도 모듈이 담당.

**중요도 근거:** ★★★ — 이 슬라이드(p7)가 **강의 전체의 분류 축**이다. 이후 등장하는 모든 모델은 "이 모델은 control loop의 어디까지 먹는가 / VLA인가 VLM+제어인가"로 위치가 결정된다. PaLM-E가 "VLA가 아니다"라고 굳이 강조되는 것도(§5), 마지막 §12 논쟁도 모두 이 틀의 적용.

**주의·함정:**
- **VLA ≠ VLM.** VLM(Vision-Language Model)은 입력(이미지+텍스트)을 받아 **텍스트**를 출력. VLA는 거기에 **Action 출력**까지 더한 것. 이 구분이 강의 내내 반복되는 핵심 축이다.

## 3. RT-1 (Google) — 본격 로보틱스 트랜스포머 ★★★

> "RT-1: Robotics Transformer for real-world control at scale" — Google, Everyday Robotics (Brohan et al., 2022). **슬라이드 11페이지(p8–18)를 할애** → 분량상 이번 강의에서 가장 자세히 다룬 모델이자, 부품 하나하나에 **필기가 가장 많이 달린** 모델.

**동기 (p9):**
- ML의 발전은 **거대 데이터셋 + 표현력 있는 모델**이 견인해 왔다.
- 그런데 로보틱스는 **대규모·다양한 로봇 데이터의 부족**으로 뒤처져 있었다.
- 실시간으로 동작하며 **효과적으로 일반화**하는 모델이 필요.

**RT-1 개요 (p9):**
- 로봇 입력(카메라 이미지, task instruction)을 **토큰화**하고 **action을 출력**하는 멀티태스크 모델.
- **13대의 로봇으로 17개월간 수집한 130,000개 실세계 에피소드, 700개 이상의 task**로 학습 (p10: 평가 3000회 실세계 trial).
- 새로운 task·환경·물체에 대한 **zero-shot 일반화** 개선을 입증.

**모델 아키텍처 (p11) — 세 가지 핵심 부품:**
- **Image Tokenization:** ImageNet 사전학습된 **EfficientNet-B3**로 이미지를 토큰으로 평탄화.
- **Action Tokenization:** 로봇 action(팔·베이스 움직임)을 **이산 토큰으로 discretize**.
- **Token Compression:** **TokenLearner**로 적응적 토큰 압축 → 추론 속도 **2.4배** 향상.

**Action의 구성 (p12) — 반드시 기억할 표:**
> Action 형식: `[mode: arm_base] [x, y, z, roll, pitch, yaw, gripper]`

| Mode | 제어 대상 | 설명 |
|------|-----------|------|
| **arm** | 로봇 팔 | end-effector의 위치·자세·그리퍼 상태 제어 |
| **base** | 모바일 베이스 | 전진·후진·횡방향 이동 제어 |
| **arm_base** | 팔 + 베이스 | 둘을 동시 제어 |
| **noop** | 없음 | 아무 동작 안 함(대기) |

**Ground Truth 예시 (p13):** 한 에피소드는 `instruction`(예: "pick up the blue cup and place it on the coaster") + `images`(프레임 시퀀스) + `action`(ee_position, ee_rotation 쿼터니언, gripper_closed) + `robot_state`(joint position/velocity) + `metadata`(camera, success)로 구성. → **"언어 명령 + 이미지 시퀀스 → action"이라는 VLA의 데이터 한 줄**이 무엇인지 구체적으로 보여주는 슬라이드.

**중요도 근거:** ★★★ — ① 분량 최대(11p), ② 부품별 필기 밀도 최대(아래 §4), ③ 이후 RT-2·OpenVLA·Octo가 모두 RT-1의 "이미지 토큰화 + action 토큰화 + 트랜스포머" 구조를 변주·개선하므로 **계보의 뿌리**.

## 4. RT-1의 핵심 부품 — FiLM / USE / TokenLearner / Action 토큰화 ★★★

> 이 세 부품(p14–17)은 **슬라이드마다 한국어 필기가 빽빽하게 달린** 부분 → 학생이 가장 공들여 이해한 지점. 필기를 그대로 보존한다.

### 4-1. 이미지 토큰화 & FiLM (p14–15) ★★★

**구조 (p14):** 입력 이미지를 토큰 시퀀스로 변환. **FiLM + EfficientNet 기반**(RT-1은 FiLM + MBConv = Mobile Inverted Bottleneck Convolution).

**FiLM: Feature-wise Linear Modulation (p15):**
- 출처: "Visual Reasoning with a General Conditioning Layer" (Perez et al., AAAI 2018).
- **언어 정보를 시각 피처에 주입하는 단순하지만 효과적인 방법.**
- 처리: 언어 임베딩으로부터 **채널별 scale($\gamma$)·shift($\beta$)**를 계산 → 각 피처 채널에 선형 변조 적용.
- 연산: $\text{FiLM}(F) = \gamma \cdot F + \beta$

**학생 필기 (p15, 그대로 보존) — 이 슬라이드의 핵심:**
> - "이미지의 인코딩 layer의 피처들에 대해 **감마만큼 스케일링, 베타만큼 시프트**를 모든 레이어에서 수행. 그리고 그 **레이어마다의 감마·베타 값을 학습**하는 것!"
> - "**bn(batch normalization):** batch 단위 데이터가 들어오면 그들의 분산·평균을 학습해 곱하기·더하기만으로 **covariant shift 방지**" → FiLM이 BN과 유사한 메커니즘이라는 메모.
> - "이런 방식으로 **cross attention**의 작업을 했었음 / 오늘날에는 **cross attention을 쓰면 해결**되는 것"
> - **왜 필요한가(필기의 핵심 통찰):** "이전에는, **동일한 image라도 instruction이 다르면 최종 classifier 값이 달라야** 하므로, 이 **text instruction을 image encoding 과정에 영향을 미치게** 한다! (lora와 비슷한 방식). 영향 주는 방식은 batch normalization과 유사."

→ **요점:** FiLM은 "같은 이미지라도 명령이 다르면 다르게 봐야 한다"를 구현하기 위해, **언어로 시각 인코딩 자체를 조건화**하는 장치. (오늘날이라면 cross-attention으로 대체할 부분이라는 시대적 맥락 메모.)

### 4-2. Instruction Encoder — USE (p16) ★★

- RT-1은 **USE(Universal Sentence Encoder, Google)**로 instruction을 인코딩 → **512차원** 문장 임베딩.
- 특징: 단어가 아닌 **문장 단위 인코딩**, transfer learning 가능. 아키텍처는 Transformer 기반 / DAN(Deep Averaging Network) 기반 두 버전.
- **학생 필기:** "이전에(RT-1)는 구글이 만든 **USE 사용**하여 instruction encoding / **T5 : 오늘날**" → 지금은 T5 계열로 대체된다는 시대 메모.

### 4-3. TokenLearner — "인코딩을 인코딩" (p17) ★★★

- **토큰 압축 최적화:** Vision-Language 결합 토큰을 **81개 → 8개**로 압축 → 인코더 출력 처리 **2.4배** 가속.
- **선택적 정보 응축:** 공간·의미 정보를 포함한 중요한 토큰 조합을 **soft-select**, 환경 변화에 민감한 토큰을 우선 추출.
- **학생 필기 (그대로 보존):**
  > - "**인코딩을 인코딩!**" (← TokenLearner의 본질을 한마디로)
  > - "의미론적으로는 인코더에서 나온 출력들에서 **실제 액션 수행에 필요한 정보를 선별**하는 역할"
  > - "EfficientNet 결과로 나온 token은 너무 많다 → 학습이 오래 걸리므로 정보를 선별·compression → 학습을 통해 **8개의 출력으로 줄인다**"
  > - action 구성 메모: "arm: 로봇팔을 위한 action / base: wheel base → xyz / mode: ..."

**중요도 근거:** ★★★ — RT-1 부품 중에서도 **필기가 가장 밀집**(FiLM·TokenLearner). 특히 FiLM의 "언어로 시각 인코딩을 조건화한다"와 TokenLearner의 "인코딩을 인코딩(토큰 선별·압축)"은 학생이 직접 핵심을 메모로 정리한 부분 → 강조 신호 강함.

**주의·함정:**
- FiLM/USE/TokenLearner는 모두 **"그 당시(2022)의 선택"**이고, 필기가 반복해서 짚듯 **"오늘날에는 cross-attention / T5로 대체"**된다. RT-1을 절대적 정답 구조가 아니라 **계보의 출발점**으로 볼 것.

## 5. PaLM-E (Google) — Embodied Multimodal LLM ★★

> "PaLM-E: An Embodied Multimodal Language Model" (Driess et al., ICML 2023). p19–23.

**개요 (p19–20):**
- **PaLM(LLM) + Embodiment.** PaLM(거대 언어모델)에 **비전·센서 인코더**를 결합.
- 멀티모달 입력 지원: **언어 + 시각 맥락 + 로봇 상태**.
- 출력: **자연어 계획(plan)·설명·추론**. 새 task에 zero-shot 일반화.

| | PaLM | PaLM-E |
|--|------|--------|
| Input | Text | Text + Image + Robot sensors |
| Output | Text | Text (plans, explanations, reasoning) |
| Purpose | 일반 언어 이해 | 로보틱스용 언어+지각 추론 |

**RT-1과의 비교 — ★★ 가장 중요한 슬라이드 (p21):**
> **Note. PaLM-E는 VLA 모델이 아니다.**
> PaLM-E는 텍스트 기반 로봇 명령을 실제 action으로 바꾸려면 **추가 정책 모듈(control block)이 필요**하다.

- **학생 필기 (p21, 보존):**
  > - "**text instruction encoding 부분도 대체**! 즉, 기존 CNN 기반 방식을 다 걷어냈다!"
  > - "비주얼 인코딩 부분을 **ViT로 대체**"
  > - "하지만 ViT도 트랜스포머이기에 **PaLM과 두 개의 트랜스포머를 사용** → 따라서 **모델이 무겁다**"
  > - "또한 PaLM은 **VLA가 아니라 VLM**이기에 **텍스트로 뭘 할지 알려줌(action이 아님)** → 따라서 **추가적 아키텍처가 붙어야 함**"

**구체 예시 (p22):** "First, grasp the yellow block. Then, move the blue block closer. Finally, pick up the blue block." 같은 자연어 계획을 **구조화된 task 리스트(JSON: step/action/object/modifier)**로 쪼갠다.
- **학생 필기 (p22):**
  > - "**RT-1:** 이 물체를 집어, 서랍을 열어 등 **명확하고 짧은 시간 내 수행하는 구체적 task**를 수행"
  > - "**PaLM-E:** 특정 작업을 주면 **스텝별로 무엇을 해야 할지 알려주는 역할** (language model의 역할)"

**Appendix — Pa? 네이밍 (p23):** **Pa**=Pathway, **L**=Language, **I**=Image, **M**=Model. → PaLM / PaLM-E / PaLI / PaLI-X 계열 구분.

**중요도 근거:** ★★ — 분량(5p)과 필기 모두 충실하지만, 핵심은 **"PaLM-E는 VLA가 아니라 high-level planner(VLM+제어 전략, §2의 전략 2)"**라는 위치 규정. RT-1(action까지)과 PaLM-E(계획까지)의 **역할 대비**가 take-away.

**주의·함정:**
- PaLM-E는 **planning(스텝 분해)까지만** 한다. 실제 action 생성은 별도 모듈(예: RT-1 같은 low-level policy)이 받아야 한다. **"VLM은 무엇을 할지 말해주고, action은 다른 모듈"** = §2 전략 2의 전형.

## 6. RT-2 (Google DeepMind) — 웹 지식을 제어로 전이, "VLA" 등장 ★★★

> "RT-2: Vision-Language-Action Models Transfer Web Knowledge to Robotic Control" (Brohan et al., 2023). p24–28. **여기서 "VLA"라는 명칭이 본격 등장.**

**동기 (p24):**
- 기존 로봇 모델은 **로봇 전용 데이터로 물리적 action만** 학습 → **웹 스케일에서 배운 시각·의미 개념을 활용하지 못함.**

**PaLM-E 대비 개선점 (p24) — 핵심:**
- **From planning to acting:** 계획만 하던 것에서 **실제 action 생성**으로.
- **이산 action vocabulary** 도입 → 행동을 grounding.
- **VLM(웹) 데이터 + 로봇 데이터를 함께(co-finetune) 학습** → 일반 지식 + 제어 능력 결합.
- **추가 planning 모듈 없이** 물리 로봇에 바로 쓸 만큼 **lightweight·direct.**

**아키텍처 개요 (p25):**
- **VQA(Visual Question Answering)와 유사한 형식**으로 로봇 action을 학습. 백본은 **PaLM-E, PaLI-X**.
- 즉, "이미지+질문 → 답"을 "이미지+명령 → **action 토큰**"으로 푸는 방식. action을 **텍스트 토큰처럼** 출력하므로 VLM의 웹 지식이 그대로 전이된다.

**RT-1 vs RT-2 (p28, 학생 필기 보존):**
> - "**RT-1:** specific task에 강점 (예: 녹색 불 들어올 때)"
> - "**RT-2:** **long term planning** (예: 빨간 불 — 어디로 갈지, 어떤 서랍을 열지에 대한 계획 + action) → **general한 task에서 성능**"

**중요도 근거:** ★★★ — ① **VLA라는 용어·패러다임의 정식 등장**, ② "웹 지식 전이 + co-finetuning + action을 토큰으로 출력"이라는 **VLA의 결정적 아이디어**를 담음, ③ RT-1(구체적 단일 task)과의 대비를 필기로 명확히 정리. §2 전략 1(VLA)의 대표 사례.

## 7. RT-2-X & Open-X-Embodiment 데이터셋 ★★

> "Open X-Embodiment: Robotic Learning Datasets and RT-X Models" (Vuong et al., 2023). p29.

- **대규모·다종 로봇 학습 데이터셋:** 단일 팔, 양팔(bi-manual), 4족(quadruped) 등 다양한 플랫폼·환경을 아우름.
- **전 세계 34개 로봇 연구실의 기존 로봇 데이터셋 60개를 통합.**
- 이 데이터셋으로 학습한 **RT-X는 RT-2를 크게 능가.**

**중요도 근거:** ★★ — 분량은 1p로 짧지만, "VLA는 결국 **데이터 규모·다양성** 싸움"이라는 강의 전체 동기(p9)를 데이터셋 측면에서 입증하는 마디. 이후 OpenVLA·Octo가 모두 이 Open-X 데이터를 쓰므로 연결고리.

## 8. OpenVLA — 오픈소스 7B VLA ★★

> "OpenVLA: An Open-Source Vision-Language-Action Model" (2024). Stanford·UC Berkeley·TRI·Google DeepMind·Physical Intelligence·MIT. p30–38.

**개요 (p30):**
- **완전 오픈소스, 경량 VLA.** 사전학습 VLM의 인터넷 스케일 vision-language 지식 기반.
- **RT-2-X보다 7배 작지만 더 강함**(7B 파라미터). 모델·PyTorch 학습 코드 모두 오픈.
- SOTA generalist robot policy를 표방.

**학습 데이터 (p32):**
- **Open X-Embodiment의 970k 에피소드, 27개 실제 로봇 데이터셋**으로 학습 (RT-2-X보다 15개 더 많은 데이터셋).
- 입력: 단일 이미지 + 언어 명령 프롬프트 / 출력: **토큰화된 로봇 action.**

**아키텍처 (p33–34):**
- **Visual Encoder + LLM + De-tokenizer** 통합. 구조적으로 RT-X와 유사하나 **모든 구성요소가 오픈소스.**
- **Visual Encoder가 두 인코더의 결합 (p34) — 기억할 포인트:**
  - **DINOv2** → 기하/공간 이해(dense feature 추출)
  - **SigLIP** → 의미 이해, 언어 명령과의 정렬(alignment)
  - → **"공간을 보는 눈(DINOv2) + 언어와 맞추는 눈(SigLIP)"을 합쳤다**는 것이 OpenVLA 비전 인코더의 특징.

**Action 토큰화 (p35) — 구체적으로 기억할 슬라이드:**
- 로봇 action space = **7차원**: 6-DoF delta end-effector pose $(\Delta p_x, \Delta p_y, \Delta p_z, \Delta R_x, \Delta R_y, \Delta R_z)$ + 1-DoF gripper($0$=close, $1$=open).
- 각 차원을 $[-1, +1]$로 스케일 → **255개 균일 bin으로 이산화** → 각 action을 **7개 토큰 문자열**로 표현.
- 예: raw `[0.00 0.03 -0.82 0.00 -0.14 0.57 1.00]` → tokenized `"128 132 14 128 110 201 255"`.
- → RT-2가 말한 **"action을 텍스트 토큰처럼 다룬다"**가 구체적으로 어떻게 이뤄지는지 보여주는 핵심 예시.

**중요도 근거:** ★★ — VLA를 **재현 가능한 오픈소스**로 끌어내린 이정표. 특히 **action 토큰화(p35)**와 **이중 비전 인코더(p34)**는 "VLA가 실제로 어떻게 action을 만드는가"를 가장 구체적으로 보여주는 슬라이드라 기억 가치가 높다. 다만 새 패러다임이 아니라 RT-2 계보의 오픈소스 정련판이라 ★★.

## 9. Octo — diffusion action head, finetune 친화 ★★

> "Octo: An Open-Source Generalist Robot Policy" (RSS 2024). UC Berkeley·Stanford·CMU·Google DeepMind. p39–49. **Keyword: Easy to finetune.**

**특징 (p39):** 도메인·로봇을 가로지르는 완전 오픈소스 **GRP(Generalist Robot Policy).**
- 다양한 **입력 관측**(손목 카메라, 3인칭 카메라), 다양한 **로봇 embodiment**(단일 팔, 양팔, joint position), 다양한 **task**(언어 명령, goal 이미지 등) 지원.

**토큰 구조 (p40–43) — Octo의 핵심:**
- **Task Token:** 언어 명령은 **T5-base(111M)**로 토큰화 / goal 이미지는 얕은 CNN → flatten patch → 비주얼 토큰.
- **Observation Token:** 관측(손목/3인칭 카메라)도 얕은 CNN → patch 토큰.
- **Readout Token (p41):** **action 예측을 위한 압축 표현.** 트랜스포머의 출력 "포인터" 역할 → 이 토큰 위치에서 action을 읽어낸다.
- 학습(p42)에서는 시퀀스를 한 번에, 추론(p43)에서는 timestep별로 토큰을 누적하며 진행.

**Action Head (p45) — 기억할 포인트:**
- **Diffusion Policy**에서 영감 ("Diffusion Policy: Visuomotor Policy Learning via Action Diffusion", 2023).
- action head로 **DDPM(Denoising Diffusion Probabilistic Model)** 사용 → Readout Token을 조건으로 **초기 노이즈 action을 denoise하여 최종 action 생성.**
- → RT-1/RT-2/OpenVLA가 action을 **이산 토큰**으로 뽑았다면, Octo는 **diffusion으로 연속 action을 생성**한다는 점이 차별점.

**Fine-tuning (p44, 46–49):** 유연성·확장성 강조. OpenVLA 페이지 기준 비교(p48–49)에서 single-task / multiple-task finetuning 성능 비교 제시.

**중요도 근거:** ★★ — VLA action 생성의 **두 갈래(이산 토큰 vs diffusion/연속 생성)** 중 후자를 대표. Readout token 구조와 diffusion action head가 핵심. IL 강의의 Diffusion Policy(있었다면)와도 연결되는 지점. `[보충: Diffusion Policy는 IL 계열 기법으로, Octo는 이를 VLA의 action head로 끌어옴]`

## 10. π₀ (pi-zero) — Flow Matching으로 action 시퀀스 동시 예측 ★★

> "π₀: A Vision-Language-Action Flow Model for General Robot Control" — Physical Intelligence (2024). p50–53.

**동기 (p50) — 문제의식이 명확:**
- 기존 방법은 **action 생성 주파수가 낮다(SLOW).**
- 원인: **매 timestep마다 사전학습 VLM의 forward pass가 필요**하기 때문.

**아이디어 (p50):**
- **한 스텝씩** action을 순차 생성하는 대신, **diffusion 모델로 action 시퀀스를 한 번에(동시에) 예측.**
- 사전학습 **VLM + flow matching DM(diffusion 류 생성모델)**을 통합한 아키텍처.

**모델 아키텍처 (p52):**
- **VLM 백본:** PaLI-Gemma (사전학습된 2.6B로 초기화).
- **Action Expert:** from scratch 300M.
- **Conditional Flow Matching**(diffusion 같은 생성 모델)으로 **노이즈에서 action 시퀀스를 샘플링.** ($q_t$ = joint angle.)

**중요도 근거:** ★★ — VLA의 **속도 병목(매 스텝 VLM forward)**을 정조준하고, **flow matching으로 action 청크를 동시 생성**한다는 새로운 방향을 보여줌. Octo(diffusion head)와 함께 "생성 모델 기반 action" 흐름의 최신판. 분량은 짧지만(트렌드 강의 후반부) 동기-해법이 깔끔.

## 11. 결론 — 성과 · 한계 · 과제 ★

**VLA의 성과 (p56):**
- **통합 지능:** 지각 + 자연어 이해 + embodied action을 **하나의 계산 프레임**에 통합.
- **아키텍처 혁신·효율:** 구조 설계, 파라미터 효율적 학습, 실시간 추론 가속에서 진전.
- **다양한 응용:** 휴머노이드, 자율 내비게이션, 가상 비서, HRI.
- **복잡한 순차 task 성능 향상**, **실세계 전이의 견고함.**

**한계 (p57):**
- **시스템 통합 복잡성**(VLM·action planner·계층적 컨트롤러 통합) — 단, "구조가 표준화되어 가는 중".
- **에너지·연산 요구량**이 큼.
- **미지 task로의 일반화 한계.**
- **환경 변동(어수선한/변하는 환경)에 대한 견고성 저하.**

**과제 (p58):**
- **고급 Planning & Memory:** 계층적 정책, memory-augmented network, 궤적 계획 모듈.
- **통합 백본 & Transfer Learning:** 통합 Transformer 백본, temporal alignment, **sim-to-real** 강화.
- **효율적 배포:** pruning, **LoRA adapter**, quantization-aware training, 저전력 가속기.
- **Zero/Few-shot 적응:** 구성적 일반화, few-shot meta-learning, task-agnostic pretraining.

**중요도 근거:** ★ — 트렌드 강의의 마무리 요약. 개별 항목보다 "VLA가 어디까지 왔고 무엇이 남았나"의 지형도용.

## 12. 대논쟁 — VLA vs VLM + Control ★★

> p59. §2에서 깔아둔 "두 전략"의 귀결이자 강의의 마지막 메시지.

| | **VLA (End-to-End)** | **VLM + Control (Modular)** |
|--|----------------------|------------------------------|
| 아키텍처 | 비전·언어 → action을 **단일 통합 모델**로 매핑 | **두 모듈 분리**: VLM(지각/언어) + control policy(행동) |
| 장점 | - 모달리티 간 grounding 향상(joint training)<br>- sample efficiency 잠재력<br>- compact·우아한 설계 | - 모듈러 → 해석·디버깅 용이<br>- 사전학습 VLM 통합 쉬움<br>- 컴포넌트 교체 유연 |
| 단점 | - 학습 어려움(대규모 멀티모달 데이터 필요)<br>- 해석 어려움<br>- 모듈성·재사용성 낮음 | - 파이프라인이 brittle(에러 누적)<br>- 지각↔행동 인터페이스 수작업 필요<br>- **물리를 잘 이해 못함** |
| 일반화 | 공유 표현 덕에 다양한 입력에 **더 잘 일반화**되는 경우 多 | 인터페이스 품질에 의존, 미지 task 적응 어려움 |
| 데이터 요구 | 대규모 **(image, text, action) 삼중쌍** 필요 | 대규모 vision-language 데이터 + 별도 action 데이터 활용 가능 |

> **강의자료의 결론 한 줄:** *"Eventually, everything will probably move towards VLA."* (결국 모든 것은 VLA로 수렴할 것이다.)

**중요도 근거:** ★★ — 강의 전체를 관통한 **§2의 두 전략**(VLA vs VLM+제어)을 표로 정면 비교하며 닫는 마디. PaLM-E(VLM+제어)와 RT-2 이후(VLA)의 대비가 여기로 수렴. 자료가 **VLA로의 수렴**을 명시적 결론으로 제시한 점이 take-away.

**주의·함정:**
- 두 접근은 단순 우열이 아니라 **trade-off**(통합 성능·일반화 ↔ 해석성·모듈 교체 유연성). 다만 자료의 입장은 "VLA 수렴".

## 마지막 점검

**자료 성격상의 한계 (가장 중요):**
- 이 회차는 **전사본이 없다.** 따라서 "교수님이 실제로 어떤 말·비유·강조·여담을 했는지"는 복원할 수 없다. 본 정리본의 "강조"는 전적으로 **① 슬라이드 분량, ② 슬라이드에 달린 한국어 필기의 밀도/위치**에서 역산한 것이다.
- 전사본이 확보되면, 특히 다음 지점들의 실제 강조 여부를 보강해야 한다: RT-1의 부품 선택 이유, "PaLM-E는 VLA가 아니다"를 얼마나 세게 강조했는지, 마지막 "everything will move toward VLA" 결론의 톤.

**필기(학생 메모)가 집중된 지점 = 사실상의 강조 신호:**
- **RT-1 부품들**(FiLM p15, USE p16, TokenLearner p17) — 필기 최다. 특히 FiLM의 "언어로 시각 인코딩을 조건화" + TokenLearner의 "인코딩을 인코딩".
- **PaLM-E vs RT-1 역할 구분**(p21–22) — "PaLM-E는 VLM이라 추가 아키텍처 필요", "RT-1=구체 task / PaLM-E=스텝 분해".
- **RT-1 vs RT-2**(p28) — "RT-1=specific task / RT-2=long-term planning·general".
- → 필기가 반복해 강조하는 **시대적 메모**("오늘날에는 cross-attention / T5로 대체")는, 강의가 RT-1을 **정답이 아니라 계보의 출발점**으로 다뤘음을 시사.
