## 1. Policy Gradient / A2C 복습 ★

**교수님의 설명 흐름**
본격적인 내용 전에 지난 내용을 빠르게 정리하며 이번 강의의 출발점을 잡았다. 2~3분 분량으로 가볍게 처리.

**핵심 정의 / 수식** (강의자료 기준)
$$\nabla_\theta J(\theta) = \nabla_\theta \mathbb{E}_{\pi_\theta}\left[\nabla_\theta \log \pi_\theta(s,a) \left(Q_w(s,a) - V_v(s)\right)\right]$$

- **REINFORCE (MC 방식)**: return $G_t$ 를 그대로 사용
- **Baseline Actor-Critic**: Q 값을 쓰기 위해 별도 Value 네트워크 필요
- **Advantage Actor-Critic (A2C)**: $A(s,a) = Q(s,a) - V(s)$ 를 사용

**강조 포인트 & 중요도 근거**
빠르게 지나간 복습. 이번 강의의 출발점 설정 목적이므로 ★.

## 2. A3C — 핵심 아이디어 ★★★

**교수님의 설명 흐름**
"A3C는 새로운 알고리즘이라기보다 하나의 방법론에 가깝다"고 먼저 위상을 잡았다. A가 3개이므로 A3C: **Asynchronous** Advantage Actor-Critic. 기존 A2C에 비동기화 개념을 추가한 것.

**핵심 정의 / 수식** (강의자료 기준)

> **A3C (Mnih et al. 2016)**
> - 동일한 에이전트의 N개 복사본(Worker)을 병렬로 실행
> - 각 Worker는 서로 다른 환경/시작 상태에서 샘플을 수집하고 그래디언트를 계산
> - 그래디언트를 Global(Main) Network에 전달 → Global Network가 업데이트 → 업데이트된 가중치를 Worker에 다시 복사
> - Experience Replay Buffer 불필요

**왜 비동기화인가 — Correlation 문제 해소**
- RL은 순차적 태스크이므로 데이터 간 상관관계(Correlation)가 항상 문제.
- 기존 해결책: Experience Replay (과거 데이터를 섞어 랜덤 샘플링).
- A3C의 해결책: 서로 다른 환경에 있는 에이전트들이 각자 샘플을 모아오므로, **Global Network 입장에서는 들어오는 그래디언트끼리 Correlation이 없다**.

**교수님이 쓴 비유·예시·표현**
> "교수님 표현: 다단계나, 뭐랄까… 행상? 있잖아요 행상. 서울역에 던져놓고, 종로 상가에 모아서 놔. 그러면 그날 수금한 걸 가지고 나한테 가지고 오는 그런 개념이라고 보시면 되겠습니다."

각 Worker가 자기 구역에서 수금(샘플·그래디언트)해서 본점(Global Network)에 납부하는 구조. 교수님이 이 비유를 쓴 이유는, Worker들이 각자 독립적으로 움직이면서도 결국 하나의 중앙 네트워크를 업데이트하는 흐름이 직관적으로 잘 맞기 때문.

> "교수님 표현: 타겟 네트워크 할 때랑 조금 유사한 개념이에요. 현재 밸류 네트워크를 복사해서 고정시켜놓는 것처럼, 여기선 반대로 복사본을 여러 개 만들어서 각각의 환경에서 학습시키는 방식이죠."

**강조 포인트 & 중요도 근거**
★★★ — "이 방식이 A3C의 전부입니다. 전부예요." 라고 두 번 강조. 동일한 동작 구조를 서로 다른 말로 4~5회 반복 설명함. 전사본에서 이 개념에 할애된 분량이 전체의 절반 이상.

## 3. A3C — Global Network / Worker 구조 및 동작 절차 ★★★

**교수님의 설명 흐름**
구조를 설명하고, 동작 절차를 단계별로 직접 시뮬레이션하듯 서술. "이게 됩니까?" "아시겠죠?" 등 확인 질문을 반복하며 청중이 따라오고 있는지 체크.

**구조 (강의자료 기준)**

| 구성요소 | 역할 |
|---|---|
| Global (Main) Network | 파라미터 업데이트는 오직 이 네트워크만 수행 |
| Worker (Learner, Thread) | 로컬 경험으로 그래디언트 계산 → Global에 전달 → 업데이트된 가중치 수신 |

- Global Network와 Worker 모두 Actor(Policy) + Critic(Value) 둘 다 가질 수 있음
- Worker들은 같은 네트워크지만 서로 다른 파라미터를 가짐 (비동기이므로)

**동작 절차 (강의자료 기준)**

1. **(W)** 각 Worker는 로컬 경험(에피소드)을 수집하면서 그래디언트를 **누적(Accumulate)**
2. **(W)** 일정 iteration 후 누적된 그래디언트를 Global Network에 전송
3. **(G)** Global Network는 받은 그래디언트로 자신의 파라미터를 업데이트
4. **(G)** 업데이트된 가중치를 그래디언트를 보낸 Worker에게 복사(Copy)해서 전달
5. 위 과정을 모든 Worker가 비동기적으로 반복

**교수님이 쓴 비유·예시·표현**
> "교수님 표현: 워커가 첫 번째 그래디언트를 주면 글로벌이 그걸로 업데이트하고 복사해줍니다. 두 번째 워커가 그래디언트를 주면 또 업데이트하고 복사해줍니다. 그럼 두 번째 워커는 어떤 효과가 있겠습니까? 자기 그래디언트로도 업데이트하지만, 첫 번째 워커가 업데이트한 그래디언트도 반영이 된 거잖아요. 그 세 번째는 자기, 첫 번째, 두 번째가 다 반영된 거고요."

> "교수님 표현: 워커들은 자기가 직접 획득하지 않은 샘플로부터 학습된 그래디언트까지 계속 반영이 되어서 업데이트가 됩니다. 이 방식이 A3C의 전부입니다. 전부예요."

> "교수님 표현: 게임에서 각 워커는 왼쪽 아래에서 시작하거나, 오른쪽 아래에서 시작하는 식으로 서로 다른 시작 위치에서 에피소드를 모아옵니다."

**강조 포인트 & 중요도 근거**
★★★ — "전부입니다. 전부예요." 명시적 강조. 절차를 첫 번째→두 번째→세 번째 워커로 구체적 숫자를 들어가며 3회 반복 설명.

## 4. A3C — 구현 방식 및 성능 결과 ★★

**교수님의 설명 흐름**
이론 설명을 마치고, 실제 구현과 성능 결과를 간략히 언급. 실습 프로젝트와 연결 짓는 실용적 조언도 덧붙였다.

**구현 방식**
- Worker는 **멀티스레드**로 구현하는 것이 일반적
- 예: i7 프로세서 → 최대 12 스레드 → 12개의 Worker 가능
- 네트워크로 연결된 다른 PC들과의 분산 학습에도 활용 가능 (그래디언트만 주고받으면 되므로)

**성능 결과 (강의자료 기준)**
- DQN 대비 A3C: 학습 속도 빠름, 성능도 우수
- 7개의 Atari 게임 학습 시간 비교에서도 A3C가 우수
- 하나의 CPU 스레드만 쓰는 것보다 멀티스레드 A3C가 훨씬 빠르게 수렴

**교수님이 쓴 비유·예시·표현**
> "교수님 표현: 여러분들이 캡스톤에서 강화학습으로 뭔가를 하실 때, 집에 있는 PC로 CPU 레벨에서 뭔가 할 수 있다면 그냥 하나의 스레드를 돌리는 것보다 이런 방식을 쓰시는 게 훨씬 더 성능을 높일 수 있는 방법이라는 걸 말씀드리고 싶습니다."

**강조 포인트 & 중요도 근거**
★★ — 성능 그래프와 멀티스레드 실용 조언을 언급했지만, 이론 설명보다 분량이 적고 "이건 자료 보면 된다"는 흐름으로 빠르게 처리.

## 5. GAE — TD(λ)와의 유추, n-step Advantage ★★★

**교수님의 설명 흐름**
"갑자기 GAE가 왜 나왔냐면, 지금까지 배운 알고리즘들이 실제 코드에서 이미 GAE를 쓰고 있다"고 시작. TD 크리틱과 Advantage Actor-Critic의 실제 구현이 GAE를 내부적으로 사용하기 때문에 이번에 제대로 정리한다는 맥락. TD(λ)의 흐름과 완전히 평행하게 설명함.

**핵심 정의 / 수식** (강의자료 기준)

**TD 오류 = 1-step Advantage ($\delta_t$)**
$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

> 이 $\delta_t$ 가 곧 어드밴티지다. 어드밴티지의 원래 정의는 $A(s,a) = Q(s,a) - V(s)$이지만, Q 네트워크를 별도로 두지 않기 위해 수식 전개를 통해 이렇게 등가 표현을 도출했던 것을 강조.

**n-step Advantage**
$$\hat{A}_t^{(1)} = \delta_t$$
$$\hat{A}_t^{(2)} = \delta_t + \gamma \delta_{t+1}$$
$$\hat{A}_t^{(3)} = \delta_t + \gamma \delta_{t+1} + \gamma^2 \delta_{t+2}$$
$$\hat{A}_t^{(n)} = \sum_{l=0}^{n-1} \gamma^l \delta_{t+l}$$

> n-step이 늘어날수록: 편향(Bias) 감소, 분산(Variance) 증가 — TD(λ)의 n-step return과 완전히 동일한 트레이드오프.

**교수님이 쓴 비유·예시·표현**
> "교수님 표현: TD에서 어떤 TD 타겟이 가장 좋은 TD 타겟이냐는 질문이 나왔었죠. 그래서 1-step, 2-step, 3-step이 있을 때 뭐가 제일 좋은지 모르겠으니까 다 섞어서 쓰자라고 한 게 TD(λ)였죠. 어드밴티지도 마찬가지입니다. 어드밴티지를 구할 때도 몇 스텝 어드밴티지를 쓰는 게 가장 좋냐는 같은 문제에서 비롯된 겁니다."

**강조 포인트 & 중요도 근거**
★★★ — "GAE는 TD(λ)를 어드밴티지에 적용한 것"이라는 핵심 직관을 반복해서 강조. TD(λ) 설명 때의 논리 구조를 그대로 빌려옴으로써 기존 개념과의 연결을 명확하게 구축.

## 6. GAE — 수식 전개 및 최종 표현 ★★

**교수님의 설명 흐름**
n-step Advantage들의 가중 평균을 TD(λ)와 동일한 방식으로 구성. 수식 전개를 칠판에서 직접 보여주며 step-by-step으로 설명.

**핵심 정의 / 수식** (강의자료 기준)

**GAE($\gamma$, $\lambda$)**
$$\hat{A}_t^{GAE(\gamma,\lambda)} = (1-\lambda)\sum_{n=1}^{\infty} \lambda^{n-1} \hat{A}_t^{(n)}$$

이를 전개하면:
$$\hat{A}_t^{GAE(\gamma,\lambda)} = \sum_{l=0}^{\infty} (\gamma\lambda)^l \delta_{t+l}$$

**특수 케이스**
- $\lambda = 0$: $\hat{A}_t^{GAE} = \delta_t$ (1-step TD 어드밴티지만 사용, 편향 높음·분산 낮음)
- $\lambda = 1$: $\hat{A}_t^{GAE} = \sum_{l=0}^{\infty} \gamma^l \delta_{t+l}$ (Monte Carlo 방식, 편향 낮음·분산 높음)

**수식 유도 핵심 (교수님 설명)**
2-step 어드밴티지를 $\delta$ 로 분해하면 $\hat{A}_t^{(2)} = \delta_t + \gamma\delta_{t+1}$. 이런 식으로 전개하면 GAE는 결국 모든 시점의 $\delta$ 에 $(\gamma\lambda)^l$ 만큼의 지수적 감소 가중치를 주는 것과 같다. 앞의 $(1-\lambda)$ 는 가중치의 합이 1이 되도록 정규화하는 계수.

**교수님이 쓴 비유·예시·표현**
> "교수님 표현: 람다가 0에서 1 사이라고 했을 때, 이 등비수열의 합이 1/(1-λ)가 되기 때문에 앞에 (1-λ)를 곱해준 거라고 볼 수 있겠고요. TD(λ)랑 같은 겁니다."

**강조 포인트 & 중요도 근거**
★★ — 수식 자체는 설명했지만, "GAE가 TD(λ)와 같다고 직관적으로 이해하면 된다"는 큰 그림이 더 강조됨. 수식 세부 유도는 "어려운 건 아니고 그냥 이거를 분리시키기 위해 만든 것"이라고 언급.

## 7. GAE — 실제 계산 방식 ★

**교수님의 설명 흐름**
무한대 합산은 실제로는 불가능하므로, 어떻게 실용적으로 계산하는지 설명. 이미 포워드로 하는 것이 관행이지만, 직관적으로는 역방향 누적이 더 편하다는 것을 언급.

**실제 계산 방법**
무한대 대신 에피소드 길이(예: 30 step)만큼만 계산.

역방향(backward) 누적:
- 가장 뒤 스텝의 $\delta$ 를 먼저 계산
- 앞 스텝으로 올 때마다: 이전에 계산한 값에 $\gamma\lambda$ 를 곱하고 현재 $\delta$ 를 더함
- 맨 앞 스텝은 에피소드 전체 길이의 어드밴티지를 가짐
- 맨 뒤 스텝은 1-step 어드밴티지만 가짐

$$\hat{A}_t = \delta_t + (\gamma\lambda)\hat{A}_{t+1}$$

**교수님이 쓴 비유·예시·표현**
> "교수님 표현: 마치 검색 카드를 하듯이 맨 뒤에서부터 델타를 계산하고, 그 앞에 거 계산할 때는 맨 뒤에 곱했던 거에 감마람다 한번 곱해서 더하고, 또 그 앞에 거 계산할 때는 뒤에서 계산한 거에 감마람다 한번 곱해서… 이런 식으로 뒤에서부터 앞으로 와서."

> "교수님 표현: 여러분들이 속으로 기대하는 게 '백워드-포워드 또 나오겠네' 약간 이렇게 생각하시면 맞습니다."

**강조 포인트 & 중요도 근거**
★ — 구현 세부사항으로, "이렇게 계산을 하더라"고 빠르게 정리하며 넘겼음.

**강의자료와 전사본이 어긋난 부분**
- 강의자료 슬라이드 4에 "No experience replay is needed"라고 명시되어 있으나, 전사본에서 교수님은 "A2C도 결국 Experience Replay를 쓰니까..."라고 말하다가 스스로 "조금 틀린 말이지만 이해를 돕기 위해서 과장되게 말하자면"이라고 정정함. A3C 자체는 Experience Replay가 필요 없다는 것이 강의자료 기준 공식 내용.
