## 0. 강의 한눈에 보기

### 교수님의 큰 그림 (프레이밍)
> "지금까지 배운 건 스토캐스틱 폴리시인데, 오늘은 Deterministic한 폴리시를 배운다.  
> 액션이 **객관식**이 아닌 **주관식**으로 나오게 만드는 가장 최초의, 가장 선형적인 솔루션이다."

오늘 강의 주제 (슬라이드 p.3):
- Deterministic Policy Gradient (DPG)
- Deep Deterministic Policy Gradient (DDPG)

### 커리큘럼 위치
남은 강의: DDPG → **TD3** (Twin Delayed DDPG) → **SAC**

### 다룬 개념 목록 및 중요도

## 1. Deterministic Policy의 정의와 필요성 ★★★

> ★★★ — 강의 전체의 도입이자 핵심 동기. 교수님이 **주관식/객관식 비유**, **6족 로봇 예시** 등 여러 비유를 동원해 매우 충실하게 설명. 학생 필기에도 "Continuous space에서 action을 정의할 때 많이 씀"이라고 직접 기록됨.

### 교수님의 설명 흐름

지금까지 배운 것은 모두 **Stochastic Policy**: Action에 대한 **확률 분포**를 출력.  
오늘부터 다룰 **Deterministic Policy**: State를 넣으면 **딱 하나의 Action 값**만 출력.  
표기법 자체가 달라진다 (슬라이드 p.4, p.5).

### 핵심 정의 (슬라이드 p.4–5)

| | Stochastic | Deterministic |
|---|---|---|
| 출력 | 확률 분포 $\pi(\cdot \| s)$ | 결정값 $a = \mu(s)$ |
| 표기 | $\pi_\theta(a\|s)$ — "이 state에서 이 action을 할 확률" | $\mu_\theta(s) = a$ — "이 state에 넣으면 이 action이 나온다" |

### 교수님이 쓴 비유·예시

**주관식 vs. 객관식 비유**
> "너 오늘 뭐 먹고 싶니 했을 때 '나 짜장면'이라고 하면 Deterministic이고,  
> '60%는 짜장면, 30%는 파스타...'라고 하면 Stochastic이다."

**6족 로봇 예시 — 왜 Continuous Action Space가 필요한가 (슬라이드 p.5)**
- 다리 6개 × 관절 3개 = 18개 모터
- 각 모터에 정지 / +1° / −1° 딱 3가지만 허용해도 전체 액션 스페이스 = $3^{18} = 387,420,489$
- Q-Network 마지막 레이어에서 이 모든 액션에 대한 확률을 출력해야 함 → **비현실적**
- 더 정밀한 제어(0.05° 등)가 필요하다면? 이산화할수록 액션 스페이스는 무한히 커짐
- **"이 최적 액션을 찾는 것 자체가 또 하나의 최적화 문제"**

> (학생 필기) "단순화해서 3개만 해도 Action space가 크다 → 즉 제어의 Continuous space에서는 Deterministic한 것이 정히 잘 안 나온다는 조"

### 강조 포인트 & 중요도 근거
- "비타미니스틱 폴리시가 뭔지 먼저 이해해야 오늘 수업 전체가 이해된다"는 흐름
- 주관식/객관식 비유를 두 번 반복
- $3^{18}$이라는 숫자를 직접 명시

## 2. Stochastic Policy Gradient에서 쓰던 수식은 왜 못 쓰나 ★★★

> ★★★ — DPG Gradient 유도의 전제 조건. 교수님이 "이 식은 더 이상 못 쓰겠죠"라고 명시하며 강조.

### 기존 Stochastic Policy의 Objective Function & Gradient (슬라이드 p.6)

$$J(\theta) = \mathbb{E}_{\pi_\theta}\left[\nabla_\theta \log \pi_\theta(s, a) \cdot Q_w(s, a)\right]$$

(A2C에서 배운 수식)

**Deterministic이 되면 무엇이 달라지는가?**
- Action에 대한 확률 $\pi(a|s)$가 사라짐
- → 기존 수식을 **더 이상 그대로 쓸 수 없음**
- State에 대한 분포는 남아 있음

> (학생 필기) "만약 deterministic을 다 하면 더 이상 학들이(확률이) 아니기에 위의 꼴로 나타낼 수 없음"

## 3. Discounted State Distribution ★★

> ★★ — 교수님이 10분 이상 할애해 설명했지만, **"이거 자체가 DDPG를 쓰는 데 그렇게 중요하지 않다"** 고 직접 언급. 수학적 엄밀성 맥락. 그러나 슬라이드 p.7–8 전체가 이 개념에 할애됨.

### 핵심 정의 (슬라이드 p.7)

$$\rho^\mu(s') = \int_\mathcal{S} \sum_{k=1}^{\infty} \gamma^{k-1} \, p(s \to s', k, \mu) \, \rho_0(s) \, ds$$

- $\rho_0(s)$: 초기 State 분포
- $p(s \to s', k, \mu)$: State $s$에서 시작해 정책 $\mu$를 따라 $k$번 만에 $s'$에 도착할 확률
- **Stationary State Distribution**: $k$번 만에 도달하는 모든 경우를 **동일 가중치**로 더함
- **Discounted State Distribution**: $k$번 만에 도달하는 경우에 **$\gamma^{k-1}$을 가중치**로 곱해서 더함 → 가까이 빨리 오는 경로를 더 가중

### 교수님이 쓴 비유

**뚜껑 열기 비유**
> "게임 판 안에서 진행 중일 때 내가 뚜껑을 딱 열어서 '너 지금 어디 있니?'라고 체크하는 거다.  
> 100번 열었을 때 각 위치의 빈도 → 그게 확률이고 State Distribution이다."

**경희대 학생 비유**
> "경희대에 학생이 있을 확률 = 부산에서 온 애들 + 서울에서 온 애들 + ... 다 더한 것.  
> 서울에서 경희대 오는 방법도: 버스 타고 / 지하철 타고 / 걸어오는 거 다 더한 게  
> 서울에서 여기 올 확률."

### 왜 DDPG에서 Discounted State Distribution을 쓰는가 (슬라이드 p.8)

교수님이 3가지 이유를 제시:

1. **수학적으로 맞음 (Falls Out of the Math)**  
   DDPG는 Discounted Return $\sum \gamma^t r_t$를 최적화. 그 Gradient를 계산하면 $\gamma^t$가 State Distribution에 흡수됨.

2. **Critic이 이미 Discount를 씀 (The Critic Already Discounts)**  
   Actor와 Critic이 "value"의 의미를 맞춰야 함.  
   > "Actor는 Discount를 쓰지 않는 수식을 쓰고 Critic은 Discount를 쓴다 — 서로 맞지 않는다. 그걸 맞춰주기 위해."

3. **Deterministic Policy가 $d^\mu$를 깨뜨림 (Deterministic Policies Break $d^\mu$)**  
   Deterministic policy는 같은 State에서 항상 같은 Action → Limit cycle, Absorbing state, Non-ergodic 문제 발생 가능.

> (학생 필기) "Stochastic policy가 state 분포로 나타날 때의 표현. 확률적으로는 합하지 않는 것이 옳음. Why? policy gradient에서는 reward의 discount가 들어가지 않았기 때문에 discount factor로 반영하지 않았기에. return 정의할 때는 discount factor를 사용했다면 앞에 쓰이는 state distribution 대신 반영해주는 기법."

### 주의
- "실제로 구현할 때는 그냥 샘플 뽑아서 계산하면 끝" — 이 개념을 직접 코딩할 일은 없음
- Stochastic policy에서는 Exploration이 있어 어차피 크게 관계없지만, Deterministic에서는 수학적으로 필요

## 4. Deterministic Policy Gradient (DPG) — Objective Function & Gradient ★★★

> ★★★ — "DDPG의 핵심 수식." 교수님이 ~15분 이상 Chain Rule 유도에 할애. 슬라이드 p.9–13 전체. 손으로 그린 2×2 케이스 설명 포함. "여러분들이 이미 충분히 이해할 수 있는 범위 내에서 설명이 가능하다"고 하면서도 두 번 이상 재설명.

### Objective Function (슬라이드 p.9)

$$J(\theta) = \int_\mathcal{S} \rho^\mu(s) \, r(s, \mu_\theta(s)) \, ds = \mathbb{E}_{s \sim \rho^\mu}\left[r(s, \mu_\theta(s))\right]$$

- Stochastic과 달리 Action에 대한 확률 적분이 없음
- $r(s, \mu_\theta(s))$에서 Action은 $\mu_\theta(s)$로 **결정적으로 주어짐**
- 실제 구현에서 $r$은 **Q 네트워크** ($Q^\mu(s, a)$)로 대체

### Policy Gradient — DPG Theorem (슬라이드 p.9–10)

$$\nabla_\theta J(\theta) = \int_\mathcal{S} \rho^\mu(s) \, \nabla_\theta \mu_\theta(s) \, \nabla_a Q^\mu(s, a) \, ds$$

$$= \mathbb{E}_{s \sim \rho^\mu, \, a \sim \mu_\theta} \left[ \nabla_\theta \mu_\theta(s) \cdot \nabla_a Q^\mu(s, a) \right]$$

### Chain Rule 유도 (슬라이드 p.10–13)

$$\frac{\partial Q^\mu(s, \mu_\theta(s))}{\partial \theta}
= \underbrace{\frac{\partial Q^\mu(s, \mu_\theta(s))}{\partial \mu_\theta(s)}}_{\text{Action 바뀔 때 Q 변화}}
\cdot
\underbrace{\frac{\partial \mu_\theta(s)}{\partial \theta}}_{\theta \text{ 바뀔 때 Action 변화}}$$

$$= \nabla_a Q^\mu(s, a)\big|_{a = \mu_\theta(s)} \cdot \nabla_\theta \mu_\theta(s)$$

**슬라이드 p.11에서 세 항의 의미:**
1. $\frac{\partial Q}{\partial \theta}$: Actor의 파라미터 $\theta$가 바뀔 때 Q값이 어떻게 바뀌는가 → **우리가 구하고 싶은 것**
2. $\frac{\partial Q}{\partial \mu_\theta(s)}$: Action이 조금 바뀔 때 Q값이 어떻게 바뀌는가
3. $\frac{\partial \mu_\theta(s)}{\partial \theta}$: $\theta$가 조금 바뀔 때 Action 출력이 어떻게 바뀌는가

### 교수님이 쓴 비유 — 2×2 케이스로 직관 설명 (슬라이드 p.12)

교수님이 손으로 그려서 설명한 그림:

| $\nabla_\theta \mu$ (θ↑ → Action?) | $\nabla_a Q$ (Action↑ → Q?) | 결론 |
|---|---|---|
| + | + | θ↑ → Action↑ → Q↑ → **θ 키워라** |
| + | − | θ↑ → Action↑ → Q↓ → **θ 줄여라** |
| − | + | θ↑ → Action↓ → Q↓ → **θ 줄여라** |
| − | − | θ↑ → Action↓ → Q↑ → **θ 키워라** |

> "그러니까 θ 값이 커지면 Q 값이 커져라고 얘기할 수 있는 건,  
> 밑에 두 개의 식 — θ 변할 때 Action이 어떻게 변하고, Action 변할 때 Q가 어떻게 변하는지 — 이 두 개로 나눠서 표현할 수 있기 때문이다."

### DPG Algorithm 예시 (슬라이드 p.14)

On-policy 버전 예시:

$$\delta_t = R_t + \gamma Q_w(s_{t+1}, a_{t+1}) - Q_w(s_t, a_t) \quad \text{(Critic, TD error)}$$

$$w_{t+1} = w_t + \alpha \delta_t \nabla_w Q_w(s_t, a_t) \quad \text{(Critic 업데이트 — Gradient Descent)}$$

$$\theta_{t+1} = \theta_t + \beta \nabla_\theta \mu_\theta(s) \nabla_a Q^W(s_t, a_t) \quad \text{(Actor 업데이트 — Gradient Ascent)}$$

> (학생 필기) "DPG — gradient descent. but Actor는 ascent. 지금은 DDPG."

### 강조 포인트 & 주의
- PyTorch에서는 autograd로 자동 계산됨. "토치로 구현하면 그냥 이 값을 미분하면 돼."
- 수식의 $\nabla$와 $\partial$ 혼용은 교수님이 "같은 의미로 썼다"고 명시
- **Actor는 Gradient Ascent** (목적함수 최대화), **Critic은 Gradient Descent** (TD error 최소화) — 부호 주의

## 5. DDPG = DPG + Neural Network + DQN ★★

> ★★ — DPG(이론)와 DDPG(구현)의 구분. "지금까지 50분 배운 거는 DDPG가 아니라 DPG"라고 교수님이 직접 명시.

### 교수님의 설명 흐름

> "DPG에서 Q와 μ는 임의의 미분 가능한 함수. 그걸 실제로 **Neural Network**로 구현한 게 DDPG. 'Deep'이 붙은 이유."

- **DPG** (Silver et al., 2014): 이론 논문
- **DDPG** (Lillicrap et al., 2015): DPG + Neural Network. 당시 DQN 기법을 적극 활용

### DDPG의 특징 (슬라이드 p.15)

| 특징 | 내용 |
|---|---|
| Model-free | 환경 모델 없이 학습 |
| Off-policy | Experience Replay 사용 |
| Actor-Critic | Policy Network + Q Network |
| DQN 기법 결합 | Experience Replay + Target Network |

> "밸류 네트워크는 그냥 DQN입니다. TD error로 학습하는 거예요."

> "DQN으로도 Deterministic policy를 낼 수 있긴 하다 (greedy). 하지만 Continuous space에서는 답이 없다."

## 6. Exploration 문제 & OU Noise ★★

> ★★ — "강화학습에서 가장 중요한 것 중 하나"라고 직접 언급. Deterministic policy에서의 특수한 어려움 강조. 슬라이드 p.16.

### 교수님의 설명 흐름

**Stochastic vs. Deterministic의 Exploration 차이**
- Stochastic: 모든 Action에 확률이 있으므로 낮은 확률의 Action도 자연스럽게 탐색됨
- Deterministic: 같은 State → 항상 같은 Action → **탐색이 전혀 없음**

> "Deterministic이라고 하는 것은 내가 10이 나왔는데 Exploration을 한다라고 치면 뭘 해야 하지?  
> 선택지의 Action space 바운더리 자체가 없기 때문에 내가 어떤 Action이 답으로 나왔는데  
> '나 이거 안 하고 이걸 할래'라고 하기가 힘들다."

### 해결책: Noise 추가 (슬라이드 p.16)

$$a = \mu_\theta(s) + \mathcal{N}$$

- 폴리시에서 나온 Action에 **노이즈 $\mathcal{N}$을 더해서** 환경에 전달
- DDPG 원 논문에서는 **OU(Ornstein-Uhlenbeck) Noise** 사용
- 노이즈가 추가된 Action은 환경 상호작용에만 사용. 학습 시 Q값 계산에는 영향 없음

> (학생 필기) "DPG에서 noise 추가가 시인분. 4 actor network"

### 주의
- TD3에서는 다른 방식의 Exploration 사용 → 다음 강의에서 비교

## 7. DDPG 4-네트워크 구조 & Soft Update ★★★

> ★★★ — "여러분들이 가장 헷갈리는 부분"이라고 교수님이 직접 명시. 슬라이드 p.17–18. **왜 Actor에도 Target Network가 필요한지** 설명하는 데 특히 시간 할애.

### 4개 파라미터 / 4개 모델 (슬라이드 p.18)

| 파라미터 | 모델 | 역할 |
|---|---|---|
| $\theta^\mu$ | $\mu(s \| \theta^\mu)$ — **Actor (Policy Network)** | State → Action 출력 |
| $\theta^{\mu'}$ | $\mu'(s \| \theta^{\mu'})$ — **Target Actor** | TD Target의 Next Action 제공 |
| $\theta^Q$ | $Q(s, a \| \theta^Q)$ — **Critic (Value Network)** | State+Action → Q값 |
| $\theta^{Q'}$ | $Q'(s, a \| \theta^{Q'})$ — **Target Critic** | TD Target의 Next Q값 제공 |

> (학생 필기) "4 Networks — 파라미터 모델 Network"

### 왜 Actor에도 Target Network가 필요한가?

교수님이 직접 질문을 던지고 잠시 생각해보게 한 후 설명:

> "폴리시 그레디언트를 할 때는 넥스트 스테이트에 대한 값을 쓰지 않는데 굳이 타겟이 왜 필요하냐?"

> **답**: "타겟 크리틱 네트워크에도 $(s, a)$가 들어가야 된다. 그 넥스트 스테이트에 대한 Action은  
> 폴리시 네트워크가 만들어줘야 한다. 타겟 크리틱은 과거의 Q값을 가지고 있는데,  
> 그 과거 Q값에 들어가는 Action도 **과거의 Actor(Target Actor)**에서 뽑는 것이 더 낫다.  
> **타겟끼리 겸상한다.**"

슬라이드 p.21 학생 필기:
> "Deterministic → Action은 $\mu$로 뽑음. 따라서 과거의 value는 과거의 Actor에서 사용하는 Action 기반 Network로, 과거의 것을 사용하기 위해 Target Network로 둔다."

### Soft Update (슬라이드 p.17, 28)

$$\theta^{Q'} \leftarrow \tau \theta^Q + (1 - \tau) \theta^{Q'}$$
$$\theta^{\mu'} \leftarrow \tau \theta^\mu + (1 - \tau) \theta^{\mu'}$$

- $\tau \ll 1$ (예: 0.001 ~ 0.01)
- DQN의 Hard Update(주기적 완전 복사)와 달리 **매 스텝마다 아주 조금씩** 업데이트
- Target Network가 천천히 변해야 학습 안정성이 보장됨

> "현재 파라미터의 가중치를 크게, 타겟 파라미터의 가중치를 매우 적게(0.99~0.999 정도) 해서 조금씩 조금씩 바꾼다."

## 8. DDPG 전체 알고리즘 ★★

> ★★ — 슬라이드 p.22–28에 걸쳐 단계별로 설명. 전체 흐름을 확인하는 역할.

### 알고리즘 (슬라이드 p.22–28)

```
[초기화]
1. Actor μ(θ^μ), Critic Q(θ^Q) 랜덤 초기화
2. Target Networks 초기화: θ^{μ'} ← θ^μ,  θ^{Q'} ← θ^Q
3. Replay Buffer R 초기화
4. Random Process N (OU Noise seed) 초기화

[매 에피소드, 매 스텝]
5. 초기 State s₁ 관측
6. 액션 선택: a_t = μ(s_t | θ^μ) + N_t  ← Behavior Policy (노이즈 포함)
7. 환경 실행 → (r_t, s_{t+1}) 수신
8. 트랜지션 (s_t, a_t, r_t, s_{t+1}) → Replay Buffer R에 저장

[학습]
9. R에서 Mini-batch N개 샘플링: {(s_i, a_i, r_i, s_{i+1})}

10. TD Target 계산:
    y_i = r_i + γ · Q'(s_{i+1}, μ'(s_{i+1}|θ^{μ'}) | θ^{Q'})
    ↑ Next Action은 Target Actor,  ↑ Next Q는 Target Critic

11. Critic 업데이트 (Gradient Descent):
    δ_t = y_i - Q(s_i, a_i | θ^Q)
    θ^Q ← θ^Q - α · ∇_{θ^Q} (δ_t²)

12. Actor 업데이트 (Gradient Ascent):
    θ^μ ← θ^μ + α · ∇_{θ^μ}μ(s_i|θ^μ) · ∇_a Q(s_i, a|θ^Q)|_{a=μ(s_i)}
    ↑ Behavior Critic에서 그레디언트 받음 (Target Critic 아님)

13. Soft Update:
    θ^{Q'} ← τθ^Q + (1-τ)θ^{Q'}
    θ^{μ'} ← τθ^μ + (1-τ)θ^{μ'}
```

### 핵심 정리 — 누가 누구와 쌍을 이루는가 (슬라이드 p.27)

| | Next State Action | Q값 |
|---|---|---|
| **TD Target 계산** (10번) | Target Actor $\mu'$ | Target Critic $Q'$ |
| **Actor 업데이트** (12번) | Behavior Actor $\mu$ | Behavior Critic $Q$ |

> (학생 필기 p.27) "같은 Target Network끼리 Next state 가져오고 → current state 때 그냥 1:1에서"

### Replay Buffer 활용 방식

> "현재 파라미터로 액션을 딱 한 번 하면 리워드와 넥스트 스테이트가 온다.  
> 그걸 은행(Replay Buffer)에 넣어두고, 학습할 때는 그 은행에서 미니배치 뽑아서 한다.  
> 내가 학습하는 동안 그 스테이트에서 기다리다가 학습 끝나면 또 한 번 액션하는 거다."
