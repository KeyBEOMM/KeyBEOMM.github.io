이 글은 (Jekyll + Chirpy 테마 기반)에서 새로운 포스트를 작성하는 방법을 알려드리기 위한 샘플 가이드입니다. 이 문서 자체도 작성 방법을 보여주는 예시이므로 포스트를 작성할 때 편하게 참고해 보세요!

## 1. 포스트 파일 생성하기

블로그 포스트 마크다운 파일은 항상 저장소의 `_posts` 폴더 안에 위치해야 합니다. 파일을 하나 만들고 이름을 아래 규칙에 맞게 지정해 주세요.

- **파일명 규칙:** `YYYY-MM-DD-원하는제목.md`
- **예시:** `2026-03-19-my-first-post.md`

파일 이름은 **소문자 영문, 숫자, 하이픈(-)** 만을 사용하는 것이 좋습니다 (이는 브라우저 URL 주소로 사용되기 때문입니다).

## 2. Front Matter (머리말) 작성하기

모든 포스트는 맨 위에 설정값인 `Front Matter`를 포함해야만 정상적으로 인식됩니다.

```yaml
---
title: 블로그 글 작성 가이드 (How to Write a Post)
description: 포스트 목록에서 보일 작은 요약글 (선택 사항)
date: 2026-03-19 00:26:00 +0900
categories: [대분류, 소분류] # 최대 2개 권장
tags: [태그1, 태그2, 태그3] # 글 성격에 맞는 다양한 태그
pin: true # 이 값을 true로 하면 메인 화면에 고정됩니다. (선택 사항)
---
```

> **Tip:**
> `date` (작성 시간) 값의 경우 한국 시간(KST)을 기준으로 `+0900`을 이어서 붙여주시는 것이 좋습니다.

## 3. 자주 쓰이는 마크다운(Markdown) 문법

글의 본문은 일반적인 마크다운 문법으로 편하게 작성하시면 됩니다. 가장 자주 쓰이는 문법들은 아래와 같습니다.

### 텍스트 강조하기
- **굵게(Bold):** `**강조할 텍스트**` → **강조할 텍스트**
- *기울임(Italic):* `*기울일 텍스트*` → *기울일 텍스트*
- ~~취소선(Strikethrough):~~ `~~취소할 텍스트~~` → ~~취소할 텍스트~~

### 코드 블록 (Code Block)
코드를 작성할 때는 백틱(파도표 키, \`) 3개를 사용하여 감싸줍니다. 시작 위치에 언어 이름을 적으면 텍스트 색상 하이라이팅 기능이 적용됩니다.

````markdown
```python
def hello_world():
    print("Gotta do what needs to be done. Hello, KeyBEOMM!")
```
````

### 이미지 삽입하기
이미지는 드래그 앤 드롭으로 넣거나, 미리 프로젝트 폴더(`assets/` 등)에 넣어둔 뒤 아래와 같이 경로를 지정합니다.

```markdown
![이미지 설명에 대한 대체 텍스트](/assets/해당이미지경로.png)
```

## 4. 로컬에서 미리보기 (테스트)

작성한 포스트가 블로그에서 어떻게 보일지 내 컴퓨터(로컬)에서 먼저 확인하고 싶다면 아래 단계를 따라주세요. (미리 Ruby 환경 세팅이 필요할 수 있습니다.)

1. 에디터 하단의 **터미널** 창을 엽니다.
2. `bundle install` 명령어를 입력해 필요한 패키지들이 설치되어 있는지 확인합니다.
3. `bundle exec jekyll serve` (또는 `bash tools/run.sh`) 명령어를 실행하여 로컬 서버를 구동합니다.
4. 웹 브라우저 주소창에 `http://localhost:4000` 을 입력하여 접속하면 내 블로그가 어떻게 나오는지 실시간으로 확인할 수 있습니다.

## 5. 작성 완료 후 배포하기

1. 포스트 작성을 마쳤다면, 변경된 파일을 Git으로 Commit 합니다.
2. 커밋 후 `git push` 로 GitHub 저장소(`main` 브랜치)에 적용합니다.
3. 약 1분~2분 뒤, 백그라운드에서 GitHub Actions가 동작하여 사이트(https://keybeomm.github.io)가 자동으로 업데이트됩니다.
