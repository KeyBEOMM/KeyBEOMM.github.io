# 필요 부품 정리

* **개체 인식 기능** : 
    1. RFID : 거리 인식 범위 5 ~ 10cm  
            - 사용 부품 : RC522 (실제 범위 2~3 cm로 예상 범위보다 작은 문제 있었음. 추후 134.2 khz RFID reader와 외부 안테나로 해결 가능)
    2. 서보모터 : 문 개폐용 모터. 문 무게 열 수 있는 모터
            - 사용 부품 : SG60
* **배식 기능** : 
    1. 모터 : 일정 토크 이상의 모터, 사료 걸림 해결을 위한 양방향 회전 가능 모터 
            - 사용 부품 :  NEMA 17HS4401S 스텝모터, TMC2209 모터 드라이버 
* **무게 측정 기능** :
    1. 로드셀 : 정확한 사료 배식 및 잔량 측정을 위한 무게 측정 부품
            - 사용 부품 : 로드셀, HX711

* **기타 부품** :
    1. 12V 5A 어뎁터
    2. MCU ESP32 V4 : 듀얼 코어로 RTOS 사용 가능
    3. 캐패시터
    4. 배럴 젝 커넥터
    5. 빵판, 터미널 블록, 커넥터, 쇼트바
    6. 컨버터(regulator)
    7. 퓨즈

# MCU 결정

ESP32 Devkit V4 

선정 이유 : 원래 각 기능을 묶어 여러 사료통을 중앙 관제 통에 꼽아 여러 반려동물을 배식할 수 있도록 만드려고 좀 더 과스펙의 ESP32를 선정했었다.

클럭 속도는 80MHz에서 최대 240MHz까지 조절 가능
520KB SRAM
4MB SPI 플래시 메모리
Wi-Fi: 802.11 b/g/n 지원
Bluetooth: v4.2 BR/EDR 및 BLE(Bluetooth Low Energy) 지원

듀얼 코어로 작동하며, 메모리 크기도 적절했다. 

하지만 핀개수 문제, 사료 공급 방식, 부피 문제, 기획적인 모순 등으로 단일화된 급식기를 제작하였다.

추후 RTOS를 활용해 직접 멀티테스킹을 할 수 있도록 개선할 예정이다.

# 회로 

각 부품 필요 전력량 및 전압 전류 확인하여 외부 전원 공급 방식 결정. 각 부품 전압에 맞춰 분기 나누기. 로드셀, RFID와 같이 노이즈에 민감한 부품들은 regulator를 통해 분기 최대한 나누기.
캐패시터와 인덕터도 최대한 잘 활용하여 노이즈 잘 잡기. 테스트 시 납땜 및 선 연결 문제로 쇼트 혹은 과전류 공급 문제가 있는 경우 많으니 퓨즈는 꼭 활용하기.

전력량에 따른 점퍼케이블 종류도 잘 선택 12v 분기의 경우 20 AWG 이상 사용.

## 회로 

* 외부 전원 12V 5A 
* 12V 병렬연결 - tmc2209 VM, 12-5v 컨버터 VIN, 12V 5A 어뎁터

5V 벅컨버터 강하 후 병렬 연결 - ESP32 5V핀, 서보모터 5V, 5V-3.3V 컨버터 VIN x 2, 12-5V 벅컨버터 VOUT 
3.3V 컨버터 강하 후 연결 - RC522 VCC
3.3V 컨버터 강하 후 연결 - HX711 VCC

공통 GND - GND 전부 다 공통 접지


### 캐패시터 용량에 따른 특성

10µF ('106'): 벌크(Bulk) 커패시터라고 부릅니다. 전압을 안정적으로 유지하고, 갑작스러운 전력 변화(부하 변동)에 대응하는 역할을 합니다. 주로 전해 커패시터나 탄탈 커패시터가 사용됩니다.

100nF ('104'): 바이패스(Bypass) 커패시터라고 부릅니다. 상대적으로 높은 주파수의 노이즈(Noise)를 제거하여 IC가 안정적으로 동작하도록 돕습니다.
