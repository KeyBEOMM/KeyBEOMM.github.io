# 프로젝트 소개

## 多묘, 多견 구별 자동 급식기
여러 마리의 반려 동물을 키우는 가정을 위한 자동 급식기

기획 의도: 여러 마리의 반려 동물을 키우는 가정의 경우, 서열이 높거나 식욕이 많은 동물이 다른 반려동물의 음식까지 뺏어 먹는 경우가 있다.

RFID tag를 사용해 각 반려동물을 구별하고 각 반려동물의 식사 환경 보장하며, 더 나아가 각 반려동물의 식습관을 지속적으로 확인할 수 있도록 한다.

주인이 부재할 경우에도 어플을 사용하여 인터넷만 연결되어 있다면 언제 어디서든 각 반려동물에게 맞춤으로 식사를 제공할 수 있다. 

로드셀을 내장하여 정확한 급식량을 제공할 수 있으며, 반려동물의 식사 후 잔량또한 파악하여 지속적인 반려동물의 식습관을 파악할 수 있다.

# 프로젝트 구상

내가 맡은 부분은 자동 급식기 내부 동작 하드웨어 설계 및 소프트웨어 개발이다.

해야할 일은 다음과 같다. 

1. 하드웨어
    - 필요 기능 구상
    - 기능에 맞는 하드웨어 설계 구상
    - 필요 부품 선정 및 부품 성능 테스트(원하는 성능이 실제로 나오는지, 크기는 디자인 하기에 괜찮은지)
    - MCU 결정 (연산 처리 속도(듀얼코어),메모리, GPIO핀 수..)
    - 회로도 구상 및 필요 전력 구상 
    - 부품 실제 테스트
    - 회로 설계 및 테스트
    - PCB 설계 및 테스트

2. 소프트웨어 
    - 개발 환경 선정 및 구축
    - 협업 플랫폼 결정 및 구축
    - 1차 코드 작성
    - 어플 구상 및 통신 프로토콜 및 필요 통신 데이터 정리
    - 코드 테스트
    - 리팩토링 및 코드 테스트
    - 반복..

# 기능 구체화

## 필수 기능

- RFID를 활용한 개체 구별 및 지정된 개체 도달 시 사료 문 개방
- 다른 개체 접근 차단 방식 - 자동 급식기 디자인과 문으로 해결
- 개체별 급식 스케쥴 관리
- 스케쥴에 맞는 정량 급식
- 반려동물 호출 기능
- 식사 후 피드백 기능 (알림 및 데이터 저장, 시각화) 
- 앱 - 내장 보드 통신

## 선택 기능

- 훈련 모드

# 피드백

추후 프로젝트 진행 시, 기획 후 기능 구체화 그리고 구현 방식을 먼저 정확히 정리하고 부품 구매(공부), 부품 실제 성능 테스트로 넘어가는 단계를 확실히 지키는 것이 중요할 듯 하다.
하드웨어를 같이 제작하는 프로젝트의 경우 자원이 제한되어 있기 때문에 리서치가 중요함!!

부품 공부도 중요함(datasheet 등..)! 우리가 필요한 구현하고자하는 기능을 확실히 알고 그에 적절한 부품 선정! (전력량, 제어 방법(라이브러리..), 부품 성능 등등)

부품이 정해지면 제어할 MCU도 정해야함! 제어할 부품의 수는 얼마나 되며, GPIO 핀 수는 괜찮은가? 

하드웨어를 제작하는 프로젝트가 처음이었기에 자세한 리서치 및 디자인, 기능 구체화 없이 부품 결정. 테스트 방식 미흡, 피드백 미흡
